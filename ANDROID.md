# Android Build Setup

The `android/` directory is generated by Expo's prebuild and kept in version control so CI/CD runners can compile without re-running interactive prompts.

## Prerequisites

- JDK 17 (`brew install --cask temurin@17` or `asdf install java temurin-17`)
- Android SDK (`expo install android-sdk`)
- Or simply use [EAS Build](https://docs.expo.dev/build/introduction/)

## Local Build

```bash
# First time only â€” generate the gradle wrapper jar
gradle wrapper --gradle-version 8.10.2

# Debug APK
cd android
./gradlew :app:assembleDebug

# Release AAB
./gradlew :app:bundleRelease
```

## Signing

- Debug uses an automatically generated `debug.keystore` (kept out of VCS).
- Release uses environment variables:
  - `ANDROID_KEYSTORE_FILE`
  - `ANDROID_KEYSTORE_PASSWORD`
  - `ANDROID_KEY_ALIAS`
  - `ANDROID_KEY_PASSWORD`

Configure them in your CI / EAS Build environment variables.

## Versioning

Defined in `android/gradle.properties`:

```properties
VERSION_NAME=1.0.0
VERSION_CODE=1
```

Bump `VERSION_CODE` for every Play Store upload; bump `VERSION_NAME` per [VERSIONING.md](../VERSIONING.md).

## Play Store submission

`eas.json` already declares a `submit.production` target that uses `./secrets/service-account-key.json`. Drop the JSON key from Google Cloud Console there before running `eas submit --platform android`.

## Skeleton files in this tree

| File | Purpose |
| --- | --- |
| `settings.gradle` | Include `:app`, autolinking via Expo plugin |
| `build.gradle` | Root-level Android plugin classpath |
| `gradle.properties` | Hermes/NewArch/ProGuard toggles + version constants |
| `app/build.gradle` | App module; debug + release variants + signing |
| `app/src/main/AndroidManifest.xml` | INTERNET + VIBRATE permissions, scheme `opencode` |
| `app/src/main/java/com/opencode/mobile/MainActivity.kt` | RN entry |
| `app/src/main/java/com/opencode/mobile/MainApplication.kt` | App entry |
| `app/proguard-rules.pro` | Keep rules for React/Hermes |
