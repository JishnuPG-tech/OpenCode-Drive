# Release Checklist

## Pre-release Checklist

- [ ] All PRs for the release are merged into `main`
- [ ] CHANGELOG.md is updated with all changes
- [ ] Version has been bumped according to semantic versioning
- [ ] All tests pass: `npm test`
- [ ] Typecheck passes: `npm run typecheck`
- [ ] Lint passes: `npm run lint`
- [ ] App builds successfully on local machine
- [ ] All smoke tests have been run on iOS Simulator
- [ ] All smoke tests have been run on Android emulator
- [ ] Release candidate has been distributed to QA team via EAS Build preview profile
- [ ] QA team has signed off on the release candidate
- [ ] Release notes are drafted

## Build Commands

### Development build
```bash
npx eas build --profile development --platform all
```

### Preview/QA build
```bash
npx eas build --profile preview --platform all
```

### Production build
```bash
npx eas build --profile production --platform all
```

### Build for single platform
```bash
npx eas build --profile production --platform ios
npx eas build --profile production --platform android
```

### Clean build (reset cache)
```bash
npx eas build --profile production --platform all --clear-cache
```

## Signing Setup

### iOS
- Certificates and profiles are managed automatically by EAS Build using the `eas credentials` command.
- The Apple Developer account should be invited as a team member.
- Use `npx eas credentials` to manage or view signing status.
- For builds targeting Apple App Store, ensure the distribution certificate and App Store provisioning profile are properly configured.

### Android
- Android App Signing is handled by Google Play App Signing.
- An upload key must be generated and provided to EAS Build for signing the APK/AAB.
- Service account key JSON file should be stored in `./secrets/service-account-key.json` (do not commit to repository).
- Access to the Google Play Console developer account is required to manage the service account.

## Play Store Submission Steps

### Pre-submission
- [ ] Ensure production build has completed successfully with `npx eas build --profile production --platform android`
- [ ] Build should produce an AAB (Android App Bundle) ready for Play Store
- [ ] Take screenshots in every required resolution using a physical device or the EAS Metadata tool

### Upload to Google Play Console
- [ ] Open Google Play Console (https://play.google.com/console)
- [ ] Navigate to the OpenCode app page
- [ ] If this is a new app: create the listing with package name `com.opencode.mobile`
- [ ] If this is an app update: go to Production > Create new release
- [ ] Upload the AAB file generated by the EAS build
- [ ] Fill out required details:
  - Release notes (what's new in this version)
  - Screenshots (mandatory for at least phone and 7-inch tablet)
  - Feature graphic
  - App icon
  - App category, tags, and content rating
- [ ] Review the Store Listing, App Content, and Pricing sections
- [ ] Submit for review

### Post-submission
- [ ] Monitor the review status in Google Play Console
- [ ] If rejected, address the issue and resubmit
- [ ] Once approved, monitor crash reports and reviews in Play Console
- [ ] Respond to user reviews if necessary

## Post-release Tasks

- [ ] Tag the release in GitHub: `git tag v$(node -e "console.log(require('./package.json').version)") && git push --tags`
- [ ] Create a GitHub Release with release notes
- [ ] Announce the release on all relevant communication channels
- [ ] Update any open issues to mention the version where they were fixed
- [ ] If a hotfix is needed, create a branch from the release tag, fix, and follow the release process from the step of version bumping
